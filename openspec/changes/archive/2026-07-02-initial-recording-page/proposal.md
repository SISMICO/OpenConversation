## Why

The product exists to help users improve spoken language through recorded conversations and AI feedback. Today the webapp is just a Vite + React scaffold with no user-facing functionality. We need a first interactive page where users can record speech, send it for analysis, and see feedback — even if the backend analysis is mocked initially. This unlocks the core user loop and gives us a concrete surface to iterate on.

## What Changes

- Replace the default Vite starter page with a single conversation recording page.
- Add browser-based audio recording using the MediaRecorder API with Start, Pause, Resume, and Stop controls.
- Display a static, non-editable topic card that will later be generated by the backend.
- After stopping, show Send and Discard actions. Discard requires confirmation.
- Send the recorded audio blob as `multipart/form-data` to `/api/analyse` on the same domain, including the detected MIME type and fixed English language marker.
- Mock the `/api/analyse` endpoint with a 10-second sleep and random feedback text so the loading UX can be built and tested end-to-end.
- Show a loading state while the request is in flight.
- Render the returned feedback text in the main content area below the controls.
- Clear previous feedback when the user starts a new recording.
- Handle microphone permission denial, missing microphone, unsupported recording, and generic API/recording errors with simple user-facing messages.
- Ensure the layout is responsive and usable on mobile devices.

## Capabilities

### New Capabilities

- `conversation-recording`: Browser audio recording controls (start, pause, resume, stop), blob handling, and reset behavior.
- `feedback-display`: Display of analysis feedback returned by the backend/mock endpoint.
- `mock-analysis-api`: Mock `/api/analyse` endpoint that simulates processing delay and returns placeholder feedback.

### Modified Capabilities

- None.

## Impact

- `webapp/src/App.tsx` and related files: new page layout and recording logic.
- New custom hook(s) for MediaRecorder lifecycle.
- New API client function for posting audio to `/api/analyse`.
- Vite dev server proxy or mock handler configuration for `/api/analyse`.
- `webapp/src/index.css` / Tailwind styling updates for the new layout.
- No backend changes required; backend contract for `/api/analyse` will be deferred until the mock is replaced.
